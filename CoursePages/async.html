Asynchronous JavaScript

Learn how to handle asynchronous operations in JavaScript using callbacks, promises, and async/await.

Introduction to Asynchronous JavaScript

JavaScript is a single-threaded language, which means it can only execute one task at a time. However, it can handle asynchronous operations using various mechanisms, allowing it to perform tasks like fetching data from an API without blocking the main thread.

Callbacks

Callbacks are functions passed as arguments to other functions, which are invoked once an asynchronous operation is complete. They were the first way to handle asynchronous operations in JavaScript.

Promises

Promises provide a more powerful way to handle asynchronous operations. They represent a value that may be available now, or in the future, or never. A promise can be in one of three states: pending, fulfilled, or rejected.

Async/Await

Async/Await is syntactic sugar built on top of promises. It allows you to write asynchronous code in a more synchronous-like fashion, making it easier to read and maintain.

// Callback
function fetchDataCallback(url, callback) {
    setTimeout(() => {
        callback("Data from " + url);
    }, 1000);
}

fetchDataCallback("example.com", (data) => {
    console.log(data);
});

// Promise
function fetchDataPromise(url) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve("Data from " + url);
        }, 1000);
    });
}

fetchDataPromise("example.com")
    .then((data) => console.log(data))
    .catch((error) => console.error(error));

// Async/Await
async function fetchDataAsync(url) {
    try {
        let data = await fetchDataPromise(url);
        console.log(data);
    } catch (error) {
        console.error(error);
    }
}

fetchDataAsync("example.com");
